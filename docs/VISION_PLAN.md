# VISION PLAN — The Narrow Highway / Concordance

*Foundation-first build plan across four arcs. Every slice is additive, sovereign, and honors the ethos. Marked CONDUIT vs NEEDS-GENERATION and SAFE vs GATED (needs a Matt decision). This is the document the engineer executes slice by slice.*

---

## The Vision (one paragraph)

The engine already **verifies and finds; it does not generate** — it hands a re-checkable receipt instead of a "trust me," runs sovereign and offline on the operator's own box, and on the front door (`ask.py`) it points to Christ and to real people on the matters that matter, never posing as the answer itself. This plan grows that conduit into a companion for a whole life without ever letting the companion become an idol: a conversation that is **one continuous chain** instead of a cold start (The Deck); a **Hitchhiker's-Guide Bible** that pronounces the original word, shows the father's own attributed words, and connects every dot from found data; a **coach that remembers you** and hands you a tamper-evident badge whose evidence *is* the worked proof; and a **bridge to real community** that leads with real people and teaches a non-reader to read. The sharpest line runs through all four: memory + a warm voice is exactly what a lonely person substitutes for real relationship, so generation stays gated and last-resort, every path points *outward and upward*, and success is measured — as a number, not a slogan — by the person needing the tool **less** over time (John 3:30).

---

## The Four Arcs

1. **CONTINUOUS THREAD / THE DECK** — the keystone. Every `/ask` exchange becomes a content-addressed, hash-chained *exchange-card*; a conversation is a resumable deck (thread); past conversations are searchable so you resume any thread you pull; notes are the verbatim exchange plus the citations `ask.py` already returns — nothing regenerated. This is the foundation the other three need: *a coach must remember you.*
2. **HITCHHIKER'S-GUIDE BIBLE** — original language + pronounce it + historical context + church-father commentary + PD parallel translations + a visual map of your notes + character charts + prophecy traces + connect-the-dots. Almost entirely conduit over data already held (30,991 WEB verses, 360,652-row Strong's concordance, Greek + Hebrew lexicons).
3. **COACH / TUTOR + BADGES** — two honest layers: Layer A (conduit) curated curricula + deterministic grading over the existing verifiers + an objective writing gauge + tamper-evident badges; Layer B (the one deferred generation fork) an open-ended tutoring voice, Matt-gated.
4. **COMMUNITY / ANTI-LONELINESS** — a bridge to real people + Christ, never a replacement: a gatherings directory as cards guarded by an honesty verifier, accessibility by speech so a non-reader can begin, and a Bible-study map for gathering. The sharpest guardrail lives here.

## Guardrails (baked into every slice)

- **Conduit, not source.** Deck notes are verbatim exchange + `ask.py`'s existing citations — nothing summarized/regenerated (enforced by construction: `threads.py` only stores what `respond()` returned). Bible verses/words/commentary are found WEB text + real lexicon fields + the father's own attributed words. Coach grading routes through existing deterministic verifiers returning a worked trail; the writing gauge never rewrites prose. Community generates nothing.
- **Points to Christ, not an idol / bridge not replacement.** History **never** edits the crisis or ultimate response (`classify()` keys only on current text). A persistent thread is a notebook, not a companion voice. Loneliness leads with real people (order enforced by test). No proactive outreach. Success = the person freer, nearer Christ and real people, needing the tool **less** — measured as a real number in `/coach/progress`, not copy.
- **Hands a receipt, not a trust-me.** Every exchange content-addressed + chained (`prev_hash`, ledger-style); `verify_thread` proves the notes are real. Every badge is a CAS+ledger receipt whose evidence is the graded trail (Ed25519 optional on top). Every gathering listing is sealed as a real connection.
- **Never launder.** No invented pronunciation audio (real Hebrew `pron` or a labeled synthesized guide). Only PD translations/commentary, each with a `source_url`. Interpretive prophecy/character links must cite a PD source or are omitted. Gathering cards point at real contactable groups via their own source; suggestions go to a human-vetted queue.
- **Sovereign / offline.** Stdlib-first JSON stores mirroring `cas.py`; env-configurable; gitignored; Web Speech is client-side; badges work without `cryptography`. Runs on a Pi/LoRa base — the warm threads-Corpus is capped/incremental so search-your-past doesn't break small hardware.
- **PII stripped at the edge.** `redact.py`/`redact.js`/`gateway.py` before any deck write, grade, or seal; reversible mapping not stored; prune/delete honor the right to be forgotten.
- **Fake no judgement.** Non-cognitive "mastery" is distinctly-typed self-attestation that can never satisfy a machine-verified badge (enforced by test). Generation, if opened, is flagged per-sentence and never presented as a found note.

## Cross-Cutting: the invariants that must be un-regressible

- **One shared crisis-invariant regression test (ship with Arc 1 / F4).** Identical crisis input returns identical fixed `_CRISIS_RESOURCES` regardless of `thread_id`, prior exchanges, or surface. Every arc adds a route (lonely-detection, coach crisis reuse, deck context injection) that touches `classify()`/`respond()` — this is the safety-critical invariant.
- **The people-first order test (Arc 4 / D1).** The loneliness bridge leads with real people before Christ/Scripture/keeping.
- **The self-attestation test (Arc 3 / C6).** A self-attestation can never satisfy an auto-graded badge requirement.
- **The write lock (Arc 1 / F1).** `threads.append` uses a lock mirroring `receipts._LEDGER_LOCK`.
- **The warm-Corpus rule (Arc 1 / F3).** One warm threads-Corpus, rebuilt/incremented on append — never per request.
- **The `generated:true` coupling (Arc 3 / C7 ↔ Arc 1).** Composed content, if ever stored in a deck, carries `generated:true` and is never a found note.

## Risks

1. **Generation-as-idol (C7 convergence).** Once `coach.compose()` exists it becomes the path of least resistance and the voice silently supplants the conduit. Mitigation: gated off, last-resort only, per-sentence flagged, never on crisis/ultimate, always ends in a real-person handoff, and John-3:30 measured before it opens.
2. **Companion-that-replaces (Arc 4 + Deck memory).** A warm, remembering, always-available voice is what a lonely person substitutes for real relationship. Mitigation: no proactive outreach, follow-up points outward not inward, honesty verifier rejects self-as-community, people-first order enforced.
3. **Over-claiming a credential (C4).** A badge can over-claim mastery, and mandatory Ed25519 would break stdlib-first sovereignty. Mitigation: copy states exactly what was verified; signing optional over the free CAS+ledger receipt.
4. **Delete across Deck + append-only ledger.** `threads.delete` + `cas.delete` exist, but a graded seal in the append-only ledger cannot fully un-exist. Honest boundary: redact-at-rest **before** sealing, so a deleted deck leaves only PII-free, un-attributable seals — a Matt decision, not an assumption.
5. **Scale on small hardware.** A full inverted index per search over a growing thread store degrades the Pi/LoRa promise (see warm-Corpus rule).
6. **Bible-arc factual traps.** Pronunciation fix in the dead path; Greek has no `pron`; interpretive links masquerading as found data — all mitigated in B1/B2/B6.
