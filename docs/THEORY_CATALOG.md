# The 100 — theories the sciences and math actually run on

The corpus for the assay. Each theory is mapped to the engine domain that can touch it, with
an **expected class** — a hint to be TESTED, never a verdict. The assay disposes.

- **✓** — a deterministic verifier can CONFIRM/BREAK checkable claims drawn from it, and seal them.
- **~** — partial: specific relations verify; the theory as a whole is not a sealable computation.
- **○** — out of scope for sealing: foundational/empirical/interpretive → map-only (RESONANCE);
  for theology, CONCORDANT — a signpost, never HOLDS.

The point of the exercise is that the honest column is often `~` or `○`, not `✓`. A framework
that seals everything is an idol. This one can say *we cannot claim that*.

# The assay — results (2026-07-08)

Every theory was run through `verify_derivation` with a representative claim: the FP-gate's
VALIDATED true packet for each `✓`/`~` theory (a real checkable claim from its domain), and the
bare proposition for each `○` theory (no computable artifact). Graded against the predicted class.

**Outcome: 100 / 100 matched their predicted class.**

| class | meaning | count | verdict returned |
|-------|---------|-------|------------------|
| ✓ FIT | a checkable claim seals | 64 | **HOLDS** (64/64) |
| ~ PART | the checkable relation seals; the whole theory does not | 26 | **HOLDS** on the relation (26/26) |
| ○ OOS | the proposition is not a computation | 10 | **INCOMPLETE** (10/10) — never a false HOLDS |

**Null tests (deliberate falsehoods) — all caught:** Newton F=ma stated wrong → **BROKEN**;
accounting identity violated → **BROKEN**; "1900 was a leap year" → **BROKEN**.

**The lesson that matters most:** the 36 `~`/`○` theories are the point. The engine confirmed
a real relation *inside* relativity, quantum mechanics, and thermodynamics — and refused to seal
relativity, evolution, or the Trinity *as propositions*. INCOMPLETE is not "false"; it is the
engine keeping silent where it has no deterministic ground. A live demonstration of this
appeared by accident: two marquee packets typed with the wrong artifact keys came back
INCOMPLETE until corrected to the validated keys, which then HELD — proof that a mis-shaped
claim is refused, and that INCOMPLETE must never be read as a negative verdict.

**One honest robustness finding (not a correctness bug):** on the first assay run, #13 (Central
Limit Theorem, statistics) read BROKEN; a second run gave HOLDS, and it held in isolation. Cause:
a cold heavy-dependency import (scipy/numpy) inside the pool's 8s wall-clock bound can, under
load, exceed it on first use → ERROR → BROKEN. It errs **safe** — toward BROKEN, never a false
HOLDS — so the 0-false-positive guarantee is untouched; but a *true* heavy-domain claim can
transiently read BROKEN on a cold worker. Clean fix (deferred, low-risk): boot-warm the heavy
scientific deps at engine start, the same pattern already used for the corpus and graph.
